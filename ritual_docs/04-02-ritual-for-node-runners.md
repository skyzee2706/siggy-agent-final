# Ritual for Node Runners

> Net-new functionality enabled for Ritual nodes and their operators.

<CardGroup cols={1}>
  <Card icon="circle-nodes" title="Node specialization" href="/whats-new/node-specialization">
    Nodes can specialize in select types of compute including AI Inference, ZK
    Proving, or TEE Code Execution, rather than servicing all operations,
    earning outsized rewards for performance.
  </Card>

  <Card icon="shield-halved" title="Guardians" href="/whats-new/guardians">
    Granular firewall system for nodes to opt-in to servicing select types of
    execution, balancing security and performance preferences, while still
    participating in consensus.
  </Card>

  <Card icon="infinity" title="Minimized re-execution" href="/whats-new/symphony">
    Minimized compute re-execution via Symphony. Nodes verify succinct proofs
    and participate in committees rather than re-execution, reducing bandwidth
    usage and improving network efficiency.
  </Card>

  <Card icon="chart-network" title="Native Infernet Integration" href="/architecture/infernet-to-chain">
    Use the expansive Infernet network as a compute job backend, delegating
    complex operations to specialized Infernet nodes. Scale compute offerings
    without requiring local resources yourself.
  </Card>
</CardGroup>
