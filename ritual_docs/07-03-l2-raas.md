## The Rollup Challenge

Modern Rollup-as-a-Service (RaaS) providers like
[Conduit](https://www.conduit.xyz/) and [Caldera](https://caldera.xyz/) extend
rollup toolkits
([Optimism OP Stack](https://docs.optimism.io/stack/getting-started),
[Arbitrum Orbit](https://arbitrum.io/orbit),
[ZKSync ZK Stack](https://docs.zksync.io/zk-stack)) by adding precompiles and
custom functionality. However, these platforms face significant challenges:

* **Pricing Complexity**: No native support for efficiently pricing new types of
  computation added via precompiles
* **Limited Expressiveness**: Restricted ability to add heterogeneous compute
  without significant modifications
* **Execution Overhead**: All nodes must execute all computation, limiting
  scalability
* **Feature Gaps**: No support for advanced features like scheduled transactions

***

## Ritual + RaaS

Ritual's architecture introduces six key capabilities that enhance L2 RaaS
platforms:

### Efficient Precompile Pricing

[Resonance](/architecture/resonance) enables:

* Dynamic pricing for any new precompile computation
* Demand-driven resource allocation
* Automatic price discovery through broker competition

### Heterogeneous Compute

Native support for diverse computation:

* Custom precompiles for specialized workloads
  ([AI Inference, ZK Proving & Verification, TEE Execution, Chain Abstraction, and more](/whats-new/evm++-sidecars/overview))
* Optimized execution with specialized hardware nodes

### Minimized Re-execution

[Symphony](/architecture/symphony)'s EOVMT protocol provides:

* Node specialization for different computation types
* Reduced replicated execution overhead
* Parallel processing of independent computations
* Enhanced scalability through distributed verification

### Scheduled Transactions

Built-in support for
[scheduled transactions](/architecture/scheduled-transactions):

* Conditional computation triggers
* Enshrined top-of-block inclusion

### EIP Extensions

Via [EVM++ EIP Extensions](/whats-new/evm++/eip-extensions/overview):

* Continual support and inclusion of in-demand EIPs that improve UX + DX
* Day-0 support for [EIP-665](/whats-new/evm++/eip-extensions/eip-665),
  [EIP-5027](/whats-new/evm++/eip-extensions/eip-5027),
  [EIP-5920](/whats-new/evm++/eip-extensions/eip-5920), and
  [EIP-7212](/whats-new/evm++/eip-extensions/eip-7212)

### Robust Blob Pricing

[Resonance](/architecture/resonance) enables:

* Efficient pricing of EIP-4844 blob data
* Dynamic adjustments based on network conditions
