# Where we're going

> Ritual's public testnet is next, with a variety of net-new functionality on its way, with a focus on performance and seamless user experience.

***

### [Symphony](/whats-new/symphony)

1. Implementing model and proof sharding for our
   [distributed verification system](/architecture/symphony#distributed-verification-via-partitioned-proofs)
2. Enhancing consensus mechanisms to improve scalability
3. Extending Symphony to
   [support aBFT protocols](/architecture/symphony#symphony)
4. Enhancing network efficiency through data encoding and optimized message
   distribution

***

### [Resonance](/whats-new/resonance)

1. Open-sourcing tooling to
   make it easier to run brokers
2. Adding
   [scheduled and streaming transaction support](/architecture/resonance#looking-forward)
3. Exploring posted price mechanisms for improved UX
4. Battle-testing Resonance
   [alongside](/architecture/resonance#transaction-lifecycle) the standard
   EIP-1559 transaction fee mechanism
5. [Decentralizing](/architecture/resonance#desired-properties) the Resonance
   auctioneer
6. Enabling cryptographic computational integrity and privacy gadgets for the
   auctioneer via zkVMs & FHE

***

### Privacy Gadgets

1. Developing novel privacy primitives (Cascade) that enables secure operations
   on AI models while protecting sensitive data
2. Incorporating existing Multi-Party Computation (MPC) and Fully Homomorphic
   Encryption (FHE) gadgets for comprehensive privacy protection to the end user

***

### Agents

1. Building the Agent Launchpad to enable easy deployment of autonomous agents
   with built-in economic incentives and security guarantees
2. Developing L2Beat-style analytics frontend to track agent performance,
   revenue, and key metrics across different protocols
3. Creating a robust agent framework that provides:
   * Standardized interfaces for agent-protocol interactions
   * Built-in safety mechanisms and circuit breakers
   * Composable agent primitives for rapid development
4. Implementing agent-specific sidecars for efficient on-chain execution of
   agent logic and state management

***

### Economic Design

1. Adding support for restaking, allowing token holders to enhance network
   security
2. Continuing to reward meaningful network participation through
   Proof-of-Useful-Work
3. Implementing security measures to penalize malicious node behavior through
   staking and slashing mechanisms

***

### [Model Marketplace](/beyond-crypto-x-ai/model-marketplace)

1. Launching an auction system for model licensing rights
2. Creating a dedicated marketplace for AI model intellectual property trading

***

### Asynchronous Execution

1. Supporting
   [long-running, stateful, asynchronous precompile execution](/architecture/execution-sidecars#looking-forward)
2. Adding the ability to
   [cancel ongoing computational tasks](/architecture/scheduled-transactions#looking-forward)
   and Infernet operations
3. Building more flexible transaction management capabilities

***

### Enhanced Infernet Node Management

1. Updating our node software to support smarter work distribution
2. Improving how nodes get compensated for their computational services
3. Creating more efficient communication channels between nodes
4. Integrating
   [Resonance as an Infernet node orchestrator](/architecture/infernet-to-chain#chain-as-an-interface)

***

### Cross-Chain Compatibility

1. Finalizing and securing our cross-chain communication contracts
2. Adapting our technology to work with non-Ethereum blockchains
3. Building comprehensive testing frameworks for cross-chain operations

***

### Expanded AI Capabilities

1. Adding support for more AI model types, including image generation models
2. Broadening the range of AI operations possible on the network

***

### Hardware Security Enhancement

1. Enabling GPU-based Trusted Execution Environments for secure model operations
2. Improving the security and efficiency of AI computations

***

### Additional Client Support

1. Adding compatibility with more Ethereum execution clients, starting with Reth
